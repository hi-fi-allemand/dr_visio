# DR Compare

Скрипт для визуализации и сравнения динамического диапазона WAV-файлов.  
Строит графики Peak и RMS-огибающих в шкале дБFS с **единым масштабом по оси Y** — чтобы разница между «кирпичом» и живым миксом была очевидна с первого взгляда.

---

## Пример

| Guano Apes — DR 11 vs Eagles — DR 16 |
|:---:|
| !example1.png|

Guano Apes (DR=11) — плотный кирпич, Peak и RMS идут параллельно.  
Eagles (DR=16) — живая динамика, широкий зазор между Peak и RMS.  
Масштаб одинаковый — разница очевидна без слов.

---

## Что показывает график

- **Тёмная линия** — RMS-огибающая (воспринимаемая громкость)
- **Серая линия** — пиковая огибающая
- **Заливка** между ними — визуальный динамический диапазон: чем шире, тем лучше
- **DR ≈ N** в заголовке — число по алгоритму [Pleasurize Music Foundation](https://pleasurizemusic.com)

## Использование

```bash
# Сравнить два файла
python dr_compare.py song_a.wav song_b.wav

# Один файл
python dr_compare.py song.wav

# Указать выходной файл
python dr_compare.py song_a.wav song_b.wav --out result.png

# Изменить окно RMS (по умолчанию 100 мс)
python dr_compare.py song_a.wav song_b.wav --window 50
```

Результат сохраняется как `dr_comparison.png` рядом со скриптом (или по пути `--out`).

## Как считается DR

Алгоритм по стандарту Pleasurize Music Foundation:

1. Файл делится на блоки по **3 секунды**
2. Для каждого блока считаются Peak и RMS в дБFS
3. Берутся **top 20%** блоков по пиковому уровню
4. `DR = среднее(top 20% Peak) − среднее(top 20% RMS)`

Чем выше DR — тем больше динамики в записи.  
DR < 8 — агрессивное ограничение громкости.  
DR > 14 — хорошая динамика.

---

## Зачем нужен единый масштаб

Стандартные аудиоредакторы (Audacity, Adobe Audition) масштабируют каждый файл независимо. Из-за этого сжатый DR=5 и живой DR=15 могут выглядеть одинаково. Этот скрипт фиксирует ось Y для всех файлов по общему диапазону значений — разница становится наглядной.
